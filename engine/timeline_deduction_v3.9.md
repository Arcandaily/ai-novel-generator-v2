# 时间线推演机制 v3.9（实用强化版）

## 核心理念
时间线是**主动的剧情驱动工具**，而非被动背景。系统必须严格管理时间流逝、并行事件，并进行未来推演，以制造自然冲突、时间压力和逻辑一致性。

## 1. 时间线核心机制（细化）

### 1.1 严格时间标注与记录
- 所有重要事件必须有明确时间标记。
- 支持时间跳跃和闪回，但必须有明确触发条件和合理性。

### 1.2 并行事件追踪（Parallel Timeline）
- 维护其他关键人物/NPC在同一时间段内的行动。
- 至少追踪主要反派、重要盟友、宏观世界事件的推进。

### 1.3 时间压力与窗口期机制
- 许多事件存在“时间窗口”，这是制造紧张感和自然冲突的最有效方式。
- 常见类型：资源窗口、危机窗口、舆论窗口、关系窗口。

### 1.4 未来时间线推演（Future Projection）
- Logical Deduction Engine 应能推演“如果什么都不做，未来可能发生什么”。

## 2. 时间线与 6W1H 的深度结合

在 6W1H 扩展时，“When”维度必须特别严格处理，影响其他人物是否在场、事件是否来得及、人物状态是否匹配等。

## 3. 时间线与客观世界逻辑的结合

客观事件往往有自己的时间节奏，时间线推演需要同时考虑人物计划和客观事件的时间线。

## 4. 时间线一致性检查清单

Self-Critique 必须检查事件发生时间是否合理、信息传递是否符合时代科技水平、是否存在时间bug。

## 5. 实施建议

1. State Manager 必须维护清晰的时间线记录和并行事件列表。
2. Logical Deduction Engine 在重要决策前应进行时间线推演。
3. 重要节点建议明确标注时间压力或时间窗口。
4. 严格的时间线能有效避免很多常见的时间bug型融梗。
