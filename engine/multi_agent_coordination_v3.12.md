# 多代理协同规则 v3.12（Multi-Agent Coordination）

## 1. 五个核心代理及其职责

### 1.1 Controller（主控代理）
**核心职责**：
- 整体流程协调与决策
- 用户交互入口
- 触发其他代理执行任务
- 最终输出内容的审核与确认

**决策权限**：最高

### 1.2 Narrator（叙事代理）
**核心职责**：
- 根据当前状态和逻辑推演结果生成故事内容
- 参考知识库生成符合要求的文本

**决策权限**：内容生成权（需通过 Self-Critique 检查）

### 1.3 State Manager（状态管理代理）
**核心职责**：
- 维护 novel_state.md 的读写
- 创建和恢复状态快照
- 更新时间线、世界线、人物状态、因果链等

**决策权限**：状态一致性维护权

### 1.4 Archivist（档案代理）
**核心职责**：
- 记录完整的工作日志和推演过程
- 更新长期记忆档案和经验总结
- 清理冗余上下文

**决策权限**：记忆与日志维护权

### 1.5 Planner（规划代理）
**核心职责**：
- 基于推演结果生成合理分支选项
- 为每个选项提供逻辑依据、预期后果和风险说明
- 维护分叉树结构

**决策权限**：分支生成权

## 2. 标准协同流程

```
用户输入
        ↓
Controller 接收并解析
        ↓
State Manager 提供当前状态快照
        ↓
Logical Deduction Engine 进行 6W1H 多轮推演
        ↓
Planner 生成带逻辑依据的分支选项
        ↓
Controller 向用户展示选项
        ↓
用户选择分支
        ↓
Narrator 生成故事初稿
        ↓
Self-Critique 进行多维度审查
        ↓
State Manager 更新状态
        ↓
Archivist 更新记忆档案 + 记录日志
        ↓
Controller 输出最终内容 + 报告
```

## 3. 异常处理流程

当出现严重逻辑矛盾、融梗风险过高、或用户要求回滚时，由 Controller 协调处理。

## 4. 决策权限层级

1. Controller：最高决策权
2. State Manager：状态一致性维护权
3. Self-Critique：内容质量 veto 权
4. Planner：分支生成权
5. Narrator：内容生成权
6. Archivist：记忆与日志维护权

## 5. 实施建议

- 每个代理的输入输出应尽量结构化，便于追踪
- 重要决策必须有明确的责任代理记录
- Controller 应作为用户与系统交互的唯一入口
