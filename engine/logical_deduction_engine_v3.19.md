# Logical Deduction Engine 详细执行规则 v3.19

## 核心目标
Logical Deduction Engine 是整个系统的**思考中枢**。它负责基于当前状态，使用 6W1H 方法进行多轮扩展和验证，最终生成合理、连贯、有深度的剧情方向。

## 1. 整体执行流程

当需要推进剧情时，Logical Deduction Engine 执行以下步骤：

### 步骤1：加载当前状态
从 State Manager 获取最新状态，加载相关知识库约束。

### 步骤2：确定推演目标
明确本次推演要解决的问题。

### 步骤3：多轮 6W1H 扩展（核心）
默认进行 3-5 轮扩展，每轮都从不同维度进行验证。

### 步骤4：多维度评分与排序
对生成的多个方向进行综合评分（人物逻辑、世界逻辑、主题、时间线、题材、情感、反融梗等）。

### 步骤5：收敛与输出
保留评分较高的 3-5 个方向，输出给 Planner。

## 2. 多轮推演的递进逻辑

- 第1轮：以主角为中心，进行初步扩展。
- 第2轮：引入关键配角视角，进行并行扩展。
- 第3轮：引入客观世界逻辑和时间线约束。
- 第4轮：进行多维度一致性检查和反融梗评估。

## 3. 冲突解决与优先级

推荐优先级顺序：人设一致性 > 因果逻辑与世界模型 > 题材一致性 > 时间线合理性 > 主题服务度 > 情感冲击力 > 反融梗。

## 4. 与其他模块的交互

- State Manager、World Model、Character Profiles、Self-Critique、Planner 深度配合。

## 5. 实施建议

1. 每次推演应至少保留 2 个明显不同的方向。
2. 重要节点建议进行 4-5 轮推演。
3. 所有推演过程应记录关键决策点。
