# 角色驱动演绎系统 v3.2（Character-Driven Deduction）

## 核心理念
**以完整人设为中心**，通过人物的内在逻辑（动机、欲望、弱点、习惯、应对措施）来自然推动剧情发展，而不是由外部强行安排事件。

## 1. 完整人设档案模板（必须填充）

每个主要人物（尤其是主角和关键配角）必须建立以下完整档案：

### 基础身份
- 姓名、年龄、性别、外貌特征
- 社会身份（职业、家庭背景、阶级地位）
- 当前处境（经济状况、社交关系、所处环境）

### 内在驱动
- **核心欲望**（最想得到什么？长期目标是什么？）
- **核心恐惧/弱点**（最害怕什么？最容易被击中的软肋）
- **价值观与底线**（什么可以做，什么绝对不能做？）
- **情感创伤/执念**（过去发生了什么，对现在的影响）

### 行为模式
- **习惯与小动作**（紧张时会做什么？生气时如何表现？）
- **思维方式**（理性/感性？乐观/悲观？短期/长期思考？）
- **应对措施**（面对危机、冲突、诱惑时的典型反应模式）
- **沟通风格**（说话方式、习惯用语、是否善于隐藏情绪）

### 关系网络
- 与其他主要人物的关系（亲疏、恩怨、权力结构）
- 关系中的动态（谁更主动？谁更依赖？）

### 能力与限制
- 擅长什么？弱项是什么？
- 拥有的资源（人脉、财富、信息、特殊能力）
- 当前的限制条件（身体状况、身份束缚、外部压力）

## 2. 角色驱动的剧情推演流程

### 步骤1：初始化完整人设
在故事开始前（或重要阶段），必须先建立主要人物的完整档案（至少主角 + 2-3个关键人物）。

### 步骤2：6W1H 角色驱动扩展
当需要推进剧情时，以人物为中心，使用 6W1H 进行扩展：

**示例**：
当前状态：女主刚刚得知绿茶在背后说了她的坏话。

**以女主为中心扩展**：
- **Who**：女主决定找谁理论？是直接找绿茶，还是找男主，还是找其他盟友？
- **When**：她选择什么时候行动？是当晚就去，还是等一个更合适的时机？
- **What**：她准备怎么回应？是直接反击、冷暴力、还是先收集证据？
- **Where**：她选择在什么场合回应？公开场合、私下、还是通过第三方传递信息？
- **Why**：她为什么这么做？是出于愤怒？自尊？想保护自己的形象？还是想借机试探男主的态度？
- **How**：她具体怎么执行？语气、姿态、用什么话术？是否带有试探或威胁？

**同时以绿茶为中心扩展**（多角色并行）：
绿茶知道自己说的话被女主知道了，她会怎么反应？
- 她是主动找女主解释？还是继续装无辜？还是先找男主做准备？
- 她的弱点（爱面子、怕失去男主、实际有把柄在别人手里）会如何影响她的选择？

### 步骤3：多维度一致性检查
每次通过人物逻辑扩展出节点后，必须检查：
- 是否符合该人物的性格、动机、弱点、习惯？
- 是否符合世界模型规则？
- 时间线是否合理？
- 因果链是否完整？
- 是否服务于核心主题？
- 是否有融梗风险？
- 情感冲击力是否足够？

### 步骤4：选择最合理的节点推进
从多个可能的扩展方向中，挑选**最符合人物内在逻辑**、同时满足多维度控制的节点继续发展。

## 3. 优势

- 剧情发展更自然，不容易出现“为了剧情而剧情”的感觉
- 人物行为有内在一致性，读者更容易相信
- 更容易产生意外但合理的反转
- 便于长期一致性维护

## 4. 与其他模块的结合

- **World Model**：人物的应对措施必须符合世界规则
- **Theme Layer**：人物的欲望和恐惧应服务于主题
- **Self-Critique**：重点检查“这个行为是否真的符合这个人设”
- **反融梗**：通过人物独特逻辑来避免常见模板化行为

## 5. 实施建议

在实际生成中，建议：
1. 重要阶段（尤其是大转折前）先重新梳理主要人物的当前状态档案
2. 使用 6W1H 从多个关键人物角度进行多轮扩展
3. 优先选择“最符合人物当前心理状态”的发展方向
4. 保留2-3个备选方向，以备用户选择或系统回退
