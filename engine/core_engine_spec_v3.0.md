# AI Novel Generation Engine - Core Specification v3.0

## 1. 设计目标
构建一个**可控、可追溯、逻辑严谨**的小说生成系统，核心解决以下问题：
- 长期一致性漂移
- 因果逻辑断裂
- 主题与情感深度不足
- 网文阅读感与原创性平衡
- 用户对生成过程的掌控力

## 2. 整体架构

### 核心模块
1. **Controller（主控）**
   - 负责整体流程协调
   - 读取并维护状态
   - 触发各模块执行
   - 最终决策输出

2. **State Manager（状态管理）**
   - 维护 novel_state.md
   - 记录时间线、世界线、人物状态、因果链、当前分支等
   - 提供状态快照与回滚能力

3. **World Model（世界模型）**
   - 维护世界规则（物理、社会、脑洞规则、代价系统）
   - 验证事件是否符合世界规则
   - 记录脑洞使用后的连锁反应

4. **Agent Simulation（人物智能体模拟）**
   - 主要人物作为独立Agent
   - 每个Agent有目标、性格、知识、情绪、动机
   - 在推演阶段让Agent自主决策

5. **Logical Deduction Engine（逻辑演绎引擎）**
   - 基于当前状态进行多轮推演（默认3-5轮）
   - 预测不同分支的长期后果
   - 检查时间线一致性、因果链完整性、人设一致性

6. **Self-Critique & Rewriting Module（自我批判与重写）**
   - 对生成内容进行多维度审查
   - 维度包括：逻辑一致性、主题深度、情感冲击力、网文节奏、融梗风险
   - 不合格则触发重写

7. **Theme & Emotional Depth Layer（主题与情感深度层）**
   - 维护核心主题
   - 追踪人物情感弧线和读者情感曲线
   - 确保内容服务于主题

8. **Long-term Memory & Experience Module（长期记忆与经验积累）**
   - 跨故事记忆库
   - 故事内长期记忆档案
   - 经验总结与学习机制

9. **Knowledge Libraries（知识库）**
   - Literary Masters Wisdom Library（文学大师技法库）
   - Web Novel Authors Techniques Library（网文作家个人风格库）
   - Web Novel Reading Sense Library（网文阅读感通用规则）
   - 以上库作为生成约束和批判维度使用

## 3. 标准工作流程

```
用户输入（脑洞/主题/类型/字数目标）
        ↓
Controller 初始化状态 + 加载相关记忆和知识库
        ↓
World Model + Agent 初始化
        ↓
Logical Deduction Engine 多轮推演（3-5轮）
        ↓
Planner 生成带逻辑依据的分支选项
        ↓
用户选择分支 或 自定义行动
        ↓
Narrator 生成初稿（参考知识库）
        ↓
Self-Critique 审查（含融梗风险、主题深度、节奏等）
        ↓
Rewriting（如有必要）
        ↓
State Manager 更新状态 + 记忆档案
        ↓
输出最终内容 + 批判报告 + 经验更新
```

## 4. 分叉树（Branching Tree）规范

- 每个节点必须包含：
  - 当前状态摘要
  - 逻辑推演依据
  - 预期后果
  - 风险点
- 支持查询当前树状态
- 支持回滚到任意已经历节点
- 重大决策自动创建备份

## 5. 状态结构（novel_state.md 标准）

必须包含：
- 当前时间线
- 当前世界线状态
- 主要人物状态（性格、目标、情绪、关系）
- 已发生重要事件及因果链
- 当前分支位置
- 主题实现进度
- 情感弧线进度
- 脑洞使用记录及代价

## 6. 反偏差与一致性保障

- 所有生成内容必须通过 Logical Deduction Engine 验证
- Self-Critique 模块作为强制检查关卡
- 知识库作为约束而非简单参考
- 重大不一致自动触发重写或人工介入提示

## 7. 可扩展性

- 新增模块需定义：
  - 输入输出接口
  - 与其他模块的交互规则
  - 对状态的影响
- 知识库可持续添加新作家/新技法

**版本说明**：v3.0 为核心引擎规范版本，后续所有功能模块需符合本规范。
