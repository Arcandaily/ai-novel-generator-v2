# AI Novel Generation Engine - Core Specification v3.1

## 1. 设计目标（保持不变）
构建一个**可控、可追溯、逻辑严谨**的小说生成系统。

## 2. 核心演绎逻辑：6W1H 扩展法

这是本次更新的核心改进。

**基本原理**：
任何一个剧情节点（事件、转折、场景），都可以用 **Who / When / What / Where / Why / How** 这六个维度来描述和扩展。

- **Who**：谁做了这件事？涉及哪些人物？
- **When**：什么时候发生的？在时间线上的位置？
- **What**：发生了什么具体事件？
- **Where**：发生在什么地点/场景？
- **Why**：为什么会发生？动机、因果、前因后果？
- **How**：是怎么发生的？过程、手段、方式？

**关键机制**：
- **相互扩展**：任何一个维度的信息，都可以向其他五个维度追问和扩展，从而填充出一个完整、严谨的剧情节点。
- **多轮推演**：Logical Deduction Engine 会使用这个方法进行多轮扩展和验证。
- **一致性检查**：每次扩展后，必须检查是否与其他已知状态（人物性格、世界规则、时间线、因果链、主题）冲突。

**示例**：
当系统决定“女主在晚宴上揭穿绿茶”这个节点时，会依次或并行扩展：
- Who：女主、绿茶、男主、在场宾客？
- When：晚宴第几个阶段？时间线上处于哪个位置？
- What：具体揭穿了什么内容？用了什么证据？
- Where：晚宴的具体场景（大厅、花园、阳台）？
- Why：女主为什么要在这个时机揭穿？动机是什么？（复仇？自保？保护他人？）
- How：女主是怎么拿到证据的？用了什么方式揭穿？（公开演讲？私下对话？当众播放录音？）

通过这个过程，系统能生成逻辑自洽、细节丰富、且符合多维度控制的剧情节点。

## 3. 整体架构（更新后）

核心模块保持不变，重点强化 **Logical Deduction Engine** 使用 6W1H 扩展法进行推演。

## 4. 多维度控制的结合

在 6W1H 扩展过程中，必须同时检查以下控制维度：
- 人设一致性（性格、动机、能力、底线）
- 时间线一致性
- 因果逻辑完整性
- 世界模型规则符合性（包括脑洞代价）
- 主题服务度
- 情感冲击力与读者预期
- 网文节奏与钩子强度
- 融梗风险（是否过于接近常见模板）

## 5. 标准工作流程（更新）

```
用户输入
        ↓
初始化状态 + 知识库
        ↓
6W1H 多轮扩展推演（Logical Deduction Engine）
        ↓
一致性与多维度控制检查
        ↓
生成带逻辑依据的分支
        ↓
用户选择
        ↓
生成初稿
        ↓
Self-Critique（含6W1H一致性复核）
        ↓
更新状态 + 记忆
```
