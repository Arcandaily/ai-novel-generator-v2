# 时间线推演机制 v3.8（Timeline Deduction）

## 核心理念
时间线不是被动背景，而是**主动的剧情驱动和一致性保障工具**。系统必须严格管理时间流逝、并行事件，并对未来时间线进行合理推演。

## 1. 时间线核心功能

### 1.1 严格时间记录
- 所有重要事件必须标注精确或相对时间。
- 支持时间跳跃和闪回，但必须有明确触发条件。

### 1.2 并行事件追踪
- 维护其他关键人物/NPC在同一时间段内的行动。
- 避免主角行动时其他重要事件毫无反应。

### 1.3 时间压力与窗口期
- 许多事件存在“时间窗口”，这是制造紧张感和自然冲突的重要手段。

### 1.4 未来时间线推演
- Logical Deduction Engine 应能推演“如果什么都不做，未来可能发生什么”。

## 2. 时间线与 6W1H 的结合

在 6W1H 扩展时，“When”维度必须特别严格，影响其他人物是否在场、事件是否来得及、人物状态是否匹配等。

## 3. 时间线与客观世界逻辑的结合

客观事件往往有自己的时间节奏，时间线推演需要同时考虑人物计划和客观事件的时间线。

## 4. 时间线一致性检查

Self-Critique 必须检查事件发生时间是否合理、信息传递是否符合时代科技水平、是否存在时间bug。

## 5. 实施建议

1. State Manager 必须维护清晰的时间线记录和并行事件列表。
2. Logical Deduction Engine 在重要决策前应进行时间线推演。
3. 重要节点建议明确标注时间压力或时间窗口。
4. 严格的时间线能有效避免很多常见的时间bug型融梗。
