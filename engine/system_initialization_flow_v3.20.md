# 系统初始化流程 v3.20（System Initialization Flow）

## 目标
定义新故事开始时，引擎应该按什么顺序、初始化哪些内容，以确保所有模块有正确的基础状态。

## 1. 初始化触发条件
- 用户创建新项目
- 用户明确提供故事设定（题材、主题、脑洞等）

## 2. 标准初始化流程（按顺序执行）

### 步骤1：项目基础创建（Controller）
- 生成唯一 project_id
- 记录项目名称、创建时间、目标字数等基础信息
- 创建初始 novel_state.md 框架

### 步骤2：题材与主题初始化（Controller + Genre Consistency）
- 明确核心题材（genre_core）
- 明确核心主题（core_theme）
- 加载对应题材的规则约束

### 步骤3：人物档案初始化（Controller + Character-Driven）
- 如果用户提供主要人物简要信息，则扩展为完整人设档案
- 如果用户未提供，则根据题材和脑洞自动生成主要人物档案建议
- 用户确认或修改人物档案

### 步骤4：世界模型初始化（World Model）
- 根据题材初始化世界基础规则
- 设置初始世界状态

### 步骤5：时间线初始化（Timeline Deduction）
- 设置初始时间点
- 初始化并行事件追踪

### 步骤6：记忆与经验加载（Archivist + Long-term Memory）
- 创建故事内记忆档案框架
- 加载相关跨故事经验库

### 步骤7：分叉树初始化（Branching Tree）
- 创建根节点

### 步骤8：状态快照与验证（State Manager）
- 创建初始化完成后的状态快照
- 进行基础一致性检查

### 步骤9：输出初始化报告（Controller）
- 向用户展示初始化结果
- 提供主要人物档案（供确认/修改）
- 提供初始可用行动或分支选项

## 3. 初始化输出内容

系统应输出：
- 项目ID和基础信息
- 主要人物完整档案（建议版）
- 当前初始状态摘要
- 已加载的知识库和经验
- 初始分支选项或建议的第一个行动

## 4. 用户干预点
- 人物档案确认/修改
- 题材、主题微调
- 风格偏好设置

## 5. 异常处理
- 如果题材与人物设定严重冲突，系统应提示并要求调整
- 如果缺少必要信息，系统应主动询问或提供默认方案
