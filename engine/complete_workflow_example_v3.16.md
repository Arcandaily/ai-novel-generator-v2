# 完整使用流程 + 示例 v3.16（End-to-End Workflow）

## 目标
展示从用户输入一个故事脑洞开始，到最终输出一章内容（或一个重要节点）的完整流程，以及每个模块具体做什么。

---

## 示例故事设定

**用户输入**：
- 类型：女频马甲文（现言）
- 核心脑洞：女主是隐形富豪千金，为了调查母亲当年的车祸而假装普通白领潜入嫌疑公司
- 核心主题：真相与伪装、复仇与救赎、自我价值

---

## 完整流程演示

### 阶段1：初始化（Controller + State Manager + Archivist）

Controller 接收用户输入后：
1. 创建新项目ID
2. 初始化人物档案、世界模型、主题核心
3. 加载相关经验库
4. State Manager 创建初始状态快照

### 阶段2：逻辑推演（Logical Deduction Engine）

使用 6W1H 从人物和世界两个维度进行多轮扩展，结合角色驱动 + 客观世界逻辑 + 题材一致性。

### 阶段3：分支生成（Planner）
生成 3-4 个带逻辑依据、预期后果和风险说明的分支选项。

### 阶段4：内容生成（Narrator）
根据用户选择生成初稿，参考知识库和题材要求。

### 阶段5：审查（Self-Critique）
使用 v3.13 检查清单进行六层审查（逻辑、题材、主题情感、角色驱动、反融梗、整体质量）。

### 阶段6：更新与输出（State Manager + Archivist + Controller）
更新状态、记忆，输出最终内容 + 报告 + 当前分支选项。

---

## 总结

整个流程的核心是：用户输入 → 初始化 → 6W1H 多轮驱动推演 → Planner 生成选项 → Narrator 生成 → Self-Critique 审查 → 更新状态与记忆 → 输出

每个环节都有明确代理负责，并通过状态和分叉树保持可追溯和可控。
