# 完整使用流程 + 示例 v3.17（详细版）

## 目标
提供一个**真正可落地的端到端演示**，包含每个阶段的具体输入、输出、决策依据和示例内容。

---

## 示例故事设定

**用户输入**：
- 类型：女频马甲文（现言）
- 核心脑洞：女主林晚是隐形豪门千金，因母亲5年前的车祸离奇死亡而隐瞒身份潜入嫌疑公司调查真相。
- 核心主题：真相与伪装、复仇与救赎、自我价值

---

## 完整详细流程演示

### 阶段1：初始化
Controller 创建项目，初始化人物档案、题材核心，加载经验库，State Manager 创建初始状态快照。

**输出示例**：
初始化完成。当前时间：女主入职第1天 上午9:00
女主当前状态：谨慎、隐忍、带着强烈执念

### 阶段2：逻辑推演
Logical Deduction Engine 使用 6W1H 从女主和绿茶两个角度进行多轮扩展，结合角色驱动 + 客观世界逻辑 + 题材一致性。

### 阶段3：分支生成
Planner 生成3-4个带逻辑依据、预期后果和风险说明的分支选项（低调观察型、主动接触型、意外反转型等）。

### 阶段4：内容生成
Narrator 根据用户选择生成初稿，参考知识库和反融梗原则。

**示例片段**：
女主林晚端着咖啡走向茶水间，故意放慢脚步……绿茶的声音从身后传来……

### 阶段5：审查
Self-Critique 使用 v3.13 检查清单进行审查，给出“轻微问题”（茶水间对话有常见模板痕迹，建议增加人物性格干扰）。

### 阶段6：更新与输出
State Manager 更新状态，Archivist 更新记忆和经验总结，Controller 输出最终内容 + 报告 + 当前分支选项。

---

## 流程总结

整个流程的核心是：用户输入 → 初始化 → 6W1H 多轮驱动推演 → Planner 生成选项 → Narrator 生成 → Self-Critique 审查 → 更新状态与记忆 → 输出

每个环节都有明确代理负责，并通过状态和分叉树保持可追溯和可控。
