# 冲突解决与决策机制 v3.21

## 核心理念
当多个维度（人物内在逻辑、客观世界逻辑、题材一致性、时间线、主题、情感、反融梗等）发生冲突时，系统需要有一套清晰、可执行的冲突解决规则。

## 1. 常见冲突类型

### 1.1 人物逻辑 vs 世界逻辑
人物很想做某事，但世界规则不允许。

### 1.2 人物逻辑 vs 题材一致性
人物行为符合性格，但严重偏离核心题材。

### 1.3 主题 vs 阅读感
为了主题而牺牲爽感，或为了爽感而偏离主题。

### 1.4 时间线 vs 其他逻辑
严格时间线会导致剧情拖沓或失去冲击力。

### 1.5 反融梗 vs 合理性
为了避免融梗而强行改变逻辑，导致事件不合理。

## 2. 冲突解决优先级规则（推荐）

1. 人设一致性（最高）
2. 因果逻辑与世界模型规则
3. 题材一致性
4. 时间线合理性
5. 主题服务度
6. 情感冲击力与网文阅读感
7. 反融梗与原创性

## 3. 冲突解决策略

### 策略A：妥协与平衡（最常用）
选择综合评分最高的方案。

### 策略B：重新推演
返回 Logical Deduction Engine 重新推演。

### 策略C：引入新变量
通过新事件或信息化解冲突。

### 策略D：用户介入
向用户展示冲突点和备选方案。

## 4. 实施建议

1. Logical Deduction Engine 在每轮推演后检查冲突。
2. Self-Critique 特别关注冲突点。
3. 重要冲突记录在状态中。
4. 可根据题材预设默认倾向。
